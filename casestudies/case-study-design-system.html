Built a design system for eManager to reduce UI confusion.

Redesigned the eManager UI foundation to improve consistency, reduce user effort, and speed up product changes.

Role UI/UX Designer & UI Developer
Duration Multi-month Project
Output Design System & Component Library

Problem Statement

The eManager product had inconsistent UI patterns, repeated components, and unclear interaction behavior across screens, which caused confusion for users and slowed down design and development work.

Why this mattered

This problem mattered because:

  • Users had to relearn patterns on different screens.
  • Similar actions looked and behaved differently.
  • Errors were easy to make due to unclear states.
  • Designers recreated the same components again and again.
  • Developers implemented UI differently each time.

This increased user friction and made the product harder to scale.

Hypothesis

What we concluded

If eManager had a clear, shared design system, users would experience a more predictable interface, and teams could move faster with fewer mistakes.


What action was taken

  • I created a centralized design system with shared rules, reusable components, and clear usage guidance.

What changed

  • Design and development decisions became consistent, faster, and easier to review.

Process

Explore

  • Audited existing screens and components.
  • Listed repeated patterns and inconsistencies.
  • Identified core UI elements used across the product.

Iterate

  • Started with basic components used most often.
  • Simplified variants to avoid unnecessary choices.
  • Tested components against real product screens.
  • Adjusted based on developer feedback.

Implement

  • Finalized components with clear states and behavior.
  • Created simple usage notes instead of long docs.
  • Helped teams migrate existing screens.

Trade-offs & Constraints

  • Avoided over-flexible components to keep usage clear.
  • Focused on current product needs, not future guesses.

Final Solution

The final solution was a practical design system that:

  • Standardized UI behavior across the product.
  • Reduced cognitive load by keeping patterns familiar.
  • Made actions easier to find and understand.
  • Prevented errors through clear states and feedback.
  • Allowed teams to build screens faster with confidence.

The system was designed to be simple, usable, and easy to maintain.

Impact & Learnings

Impact

  • Reduced UI inconsistency across eManager.
  • Faster design and development workflow.
  • Clearer user flows with fewer surprises.
  • Easier onboarding for new team members.

Learnings

  • A design system should solve real product problems, not look perfect.
  • Fewer options often lead to better usability.
  • Early developer involvement saves time later.
  • Consistency builds user trust more than visual polish.
