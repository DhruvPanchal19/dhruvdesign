Created a magic macro tool to help users send repeat messages faster.

Redesigned messaging experience to reduce repetitive typing and save time for all internal users.

Role UI/UX Designer & UI Developer
Duration Ongoing
Output Macro Tool Integrated

Problem Statement

Users had to write the same text messages over and over. This was slow and wasted time for everyone, from the support team to leadership.

Why this mattered

Writing the same content repeatedly is a poor use of time and causes frustration.

  • Support agents could make mistakes copying text.
  • It slowed down responses, especially for high-volume teams.
  • Leaders also had to repeat themselves, which reduced their efficiency.

Overall, this was a clear waste for the business and affected everyone’s productivity.

Hypothesis

What we concluded

The main problem was manual, repetitive typing. People needed a way to use saved messages with quick edits (like names or custom fields).


What we did

  • Designed and built a magic macro tool.
  • Now, users can save paragraphs as shortcuts and add changeable fields.
  • Each time, they just pick the macro, fill in the blanks, and send—no more full retyping.

Process

Explore

  • Looked at existing macro and template tools.
  • Explored a basic text snippet library vs. a smarter macro with editable fields.

Iterate

  • Rejected plain snippets—they didn’t allow quick changes (like recipient name).
  • Tested an in-line macro builder: users add variables (e.g., {{name}}) that prompt for input.
  • Iterated the UI for fastest access: dropdown in message box, keyboard shortcut, auto-focus on fields.

Implement

  • Shipped a simple, accessible dropdown to insert macros.
  • Built a template builder with required fields.

Constraints

  • Legacy messaging system meant some features (like drag-and-drop macros) were out of scope.
  • No research phase—had to move fast.
  • Some UI polish was deferred for speed.

Final Solution

Added a magic macro dropdown to the message area.

  • Users select a macro, fill in required fields, and the full message appears ready to send.
  • No more retyping; fewer errors; faster for everyone.
  • The tool became popular across teams, not just support.

Impact & Learnings

Directional Impact

  • Most users saved 80–90% of the time per repeated message.
  • Fewer mistakes in copy/paste.
  • Faster support responses and internal communications.

Design Learning

  • People want speed but also control—letting users add fields to macros made the tool much more useful.

What to improve next time

  • Add analytics to track macro usage and catch friction points.
  • Make it easier to share macros between teams.
